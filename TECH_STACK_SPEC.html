Project Overview

What Maase is, and the problem it solves

Maase is India's hyperlocal marketplace for authentic home-cooked food. It connects home cooks — primarily homemakers — with nearby customers who want fresh, daily home-cooked meals within a 3km radius. No dark kitchens, no restaurants. Real cooking from real homes.

🍱

Core insight: Indian kitchens are already active every day. Gas is on, ingredients are ready, cooking is happening for the family. One extra portion = almost no extra effort, but an entirely new income stream for the household.

Payments

Razorpay — India's leading payment gateway

💳

Order Payments

Razorpay SDK on mobile (UPI, cards, net banking). Server-side signature verification before order confirmation.

🔔

Webhooks

POST /api/payments/webhook — Razorpay notifies payment status. HMAC-SHA256 signature verification.

💸

Cook Payouts

UPI ID or bank account stored on Cook model. Weekly settlement via Razorpay Route or manual transfer (Phase 1).

Mobile Design Principles

  • Mobile-first — majority of users on Android/iOS
  • Minimum touch target: 48px — all buttons, inputs, chips
  • Input font-size ≥ 16px — prevents iOS Safari auto-zoom on focus
  • Bottom-sheet modals on mobile — native feel via CSS border-radius + align-items: flex-end
  • Tap highlight removed — -webkit-tap-highlight-color: transparent
  • 3km hyperlocal language — never generic "India", always area-specific

Unique Product Features

What makes Maase different from Swiggy / Zomato

🟢

Cook's Kitchen Live

Real cook timing shown to customer — when food was started today, not an estimate. Complete transparency from kitchen to door.

Unique to Maase
🍳

Maa's Batch

True scarcity: if she makes 2 portions, only 2 can be sold. No dark patterns, no fake countdowns. Authenticity as a product feature.

Unique to Maase
🤲

Feed a Neighbour

One-tap ₹120 donation to feed someone nearby, anonymously. Community built into the product from day one.

Social Impact
🎁

Referral System

Each chef gets a unique code (e.g. SUNITA-7823) to invite other home cooks. Stored in Supabase, shareable via WhatsApp with pre-filled message.

Growth
👤

Shareable Chef Profile

Static public URL per chef. Built on URL params — no backend needed. Shareable as a WhatsApp link to attract customers pre-launch.

Growth
❤️

Nominate Your Mom

Community-driven chef onboarding. Anyone can nominate a home cook. Maase personally reaches out + sends welcome kit worth ₹500.

Community

Development Setup

How to run every service locally

Prerequisites

Node.js 22+ pnpm expo-cli vercel-cli Supabase CLI (optional) 
# Install all workspace dependencies
pnpm install

# ── Customer App (physical device via tunnel) ──
./start-expo.sh
# or: nvm use 22 && npx expo start --tunnel

# ── API Service ──
cd services/api
npm run dev        # ts-node-dev, auto-restarts

# ── Cook Dashboard ──
cd apps/cook-dashboard
npm run dev        # Next.js dev server

# ── Marketing Website ──
cd apps/web
npm run dev

# ── Landing Page (instant deploy) ──
cd web-preview
npx vercel --prod --yes

# ── Database migrations ──
cd services/api
npx prisma migrate dev     # create + apply migration
npx prisma studio          # visual DB browser

Launch Phases

Staged rollout starting with Bangalore

0

Pre-Launch — Now

Landing page live. Collecting chef applications, nominations, and customer waitlist from Bangalore. Referral system driving organic word-of-mouth growth.

1

Phase 1 — Bangalore Pilot (5 Areas)

Koramangala, HSR Layout, Whitefield, Indiranagar, BTM Layout. Target: 20 verified home chefs, 200 customer orders. Full app on iOS + Android.

2

Phase 2 — Bangalore Scale (20 Areas)

Expand to 20 Bangalore areas. Target: 50 verified home chefs, optimize delivery logistics, introduce Cook's Kitchen Live + Feed a Neighbour at scale.

3

Phase 3 — Second City Expansion

Chennai, Hyderabad, or Pune based on waitlist demand. Replicate Bangalore playbook. Localize cuisine categories per city.
